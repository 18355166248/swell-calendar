# 2026-06-10 · Spectrum 2 外壳应用（apps/swell-calendar-s2）

## 背景

需要把现有 swell-calendar 日程界面用 **Adobe React Spectrum 2（S2）** 真实组件重建一套外壳，
对标 Claude Design 导出的 `swell-calendar.html` 视觉规范。S2 自带 `page.css`（改 `<html>` 全局背景）、
`Provider`、字体加载、parcel macro 样式系统 —— 这些是**宿主应用关注点**，与 `packages/calendar`
“零副作用样式 / `swell-calendar-` 前缀 / 不注入全局 CSS / 主题走 Store / 底层不 import React” 的纯库约束**直接冲突**。
因此 S2 这套不能塞进 `packages/calendar`，必须落在独立 app。

## 目标

- 在 `apps/swell-calendar-s2` 新建一个独立的 S2 应用（Vite + react-ts），接入既有 pnpm + turbo monorepo。
- 打通 S2 链路：`unplugin-parcel-macros` + `Provider` + `page.css` + `style()` macro，验证可渲染。
- 用 S2 控件搭外壳：顶部 日/周/月/调度/时间线 `Tabs`；新建/筛选 `Button`/`ActionButton`；
  搜索 `SearchField`；通知/设置 `ActionButton`+icon；左下小月历 S2 `Calendar`。
- 中间“资源×时间”调度网格**复用 `packages/calendar` 的 Timeline 引擎**（`workspace:*` 依赖），
  不重写拖拽/车道堆叠逻辑。
- 青绿强调色配进 Provider/主题层，**不在组件写死颜色**。

## 非目标

- 不改 `packages/calendar` 的公开 API 与 SPEC 能力边界（timeline 主题扩展另立任务，见
  `docs/archive/tasks/2026-06-10-timeline-theme.md`）。
- 不发布该 app（demo/外壳性质，非可复用库）。
- 不做移动端适配、不做数据获取层（事件数据由本 app 注入 mock）。

## 影响范围

- 代码：新增 `apps/swell-calendar-s2/**`（不触碰 `packages/calendar/src`）。
- 文档：本任务文档；如最终需要记录 app 与库的依赖关系，再补 `docs/ARCHITECTURE.md`。
- 运行时行为：新增一个可独立 `dev` 的 app，不影响既有 calendar 库与 Storybook。

## 现状

- monorepo `pnpm-workspace.yaml` 已包含 `apps/*`，新增 app 无需改 workspace。
- `packages/calendar` 已实现 Timeline/Scheduler（mobiscroll parity，拖拽移动/resize/空白创建）。
- 本机 pnpm 不在 PATH，用 `corepack pnpm`（pnpm@10.15.1）；node v24。

## 方案

分阶段：

- **P1 脚手架（不依赖设计稿）**：建 app、装 `@react-spectrum/s2` + dev `unplugin-parcel-macros`，
  `vite.config` 里 `macros.vite()` 排在 `react()` **前面**；`tsconfig` 支持 import attributes
  `with { type: 'macro' }`；`main.tsx` 引 `page.css` + `<Provider>`；渲染一个 accent `<Button>` +
  `style({padding:16})` 验证链路。
- **P2 库改造（依赖设计稿取值）**：见 `2026-06-10-timeline-theme.md`。
- **P3 外壳 + 挂载（依赖设计稿）**：S2 控件搭外壳，中间挂 `packages/calendar` Timeline，
  事件卡柔色用 S2 语义色浅变体经 `style()` 注入。

### 已知坑

- `unplugin-parcel-macros` 必须排在 react 插件之前，否则 Vite 报 `fs`/`url` 被 externalize。
- ESLint/TS 需支持 `with { type: 'macro' }` import attributes（TS 5.5.4 ≥ 5.3，OK）。

### 设计稿到位后的重定向（2026-06-10）

用户把 handoff 落到 `swell-video-download-remix/project/`（含 `swell-calendar.html` + 完整 React/CSS 原型）。
读完发现两条与原计划冲突的事实，据此调整：

1. **设计稿不是真 S2 组件**：它用自定义 CSS 变量（`spectrum-tokens.css`，oklch）手写了一套 S2 近似，
   token 文件明确写 "approximated" / "Adobe Clean 用 Source Sans 3 代替"。handoff README 的硬要求是
   **像素级还原**，且「用适合目标的技术重建，别照搬内部结构」。塞真 S2 `<Button>/<Tabs>` 反而对不上设计。
   → **决策改为忠实移植这套 CSS**，不引 S2 runtime（`page.css`/`Provider` 的全局背景会与 `--bg-app` 冲突）。
   `@react-spectrum/s2` + `unplugin-parcel-macros` 暂保留为依赖（vite macros 插件空转无害），留待后续
   「真 S2 组件替换」阶段。
2. **设计自带全部 5 个视图的静态实现**（含 scheduler/timeline），并非只缺 timeline 主题。
   原 Path B（给 `packages/calendar` 加 timeline 主题切片）在「先出像素级成品」目标下**不是最短路径**，
   暂缓；见 `2026-06-10-timeline-theme.md`（状态：暂缓，未阻塞本 app）。

实际改动（移植，全部在 `apps/swell-calendar-s2/`）：
- `src/styles/{spectrum-tokens,app,views}.css`：从 handoff 原样拷入。
- `src/data.ts`：类型化 SWELL 样例数据（resources/events/monthEvents + CAT_COLORS）。
- `src/icons.tsx`：line icons 转 TSX。
- `src/shell.tsx`：Sidebar / Topbar / MiniCalendar。
- `src/views.tsx`：TimeGrid(Day/Week) + 重叠车道布局 + Month + Scheduler + Timeline。
- `src/overlays.tsx`：Popover（rich/default/minimal）+ CreateDialog + SubBar。
- `src/App.tsx` / `src/main.tsx` / `index.html`：组合 + 加载设计 CSS + Source Sans 3/Code Pro 字体。
- 设计稿的 `tweaks-panel.jsx` 是 Claude Design 编辑器宿主工具（postMessage `__edit_mode_*`），
  **非真实功能，未移植**；固定采用其默认配置（light/seafoam/soft/segmented/rich/full/regular）。

验证结果：
- `pnpm --filter swell-calendar-s2 exec tsc --noEmit` 通过。
- 浏览器实测（5180）：scheduler / week / timeline 三视图 + rich popover 均像素级还原，
  seafoam 主题、今天高亮、now-line(14:33)、周末浅染、重叠车道布局正确，无 console error。
- `check-docs` / `check-arch` 通过。

剩余 / 后续可选：
- **真 S2 组件替换**：顶栏/侧栏控件渐进换成 `@react-spectrum/s2`（会偏离像素，需取舍）。
- **挂活引擎**：把 week/scheduler 的静态视图替换为 `packages/calendar` 的拖拽引擎，并用设计 token 重新着色
  （即原 Path B + 引擎复用，作为下一阶段）。
- handoff 目录 `swell-video-download-remix/` 仍在仓库根，是设计参考；如不入库可加 `.gitignore`。
- dark / accent / density 等设计稿可切配置当前固定为默认，如需做成真功能可加一个轻量设置面板。
